# DEVLOG

## PURPOSE

This file records factual changes made to the repository during execution.

It is the primary source of truth for what was actually implemented.

This file does NOT store discussions, reasoning, or plans.

---

## RULES

- Only record factual changes.
- Each entry must describe what was done, where, and why (briefly).
- Do not include discussions, assumptions, or long explanations.
- Do not duplicate information from other files.
- Keep entries short and precise.
- Use append-only format (do not rewrite history).
- Do not remove past entries (use archive if needed).

---

## UPDATE RULES

- Add a new entry after completing a task or meaningful change.
- Each entry must be atomic (one logical change or step).
- Prefer multiple small entries over one large summary.
- Reference affected files or areas when possible.
- If a change is reverted, record it as a new entry (do not edit old ones).

---

## FORMAT

Each entry must follow this structure:

- [DATE TIME] Short title
  - What was done
  - Where (files / modules)
  - Why (1 line max)
