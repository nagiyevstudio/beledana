# Концепт: Автономная Маркетинговая и Сейлз Экосистема

## 1. Методология: Принцип «Разбиения камня»

Система строится слоями через последовательную декомпозицию:

1. **Глыба:** Общая идея (Маркетинг + Сейлз).
2. **Крупные куски:** Функциональные блоки (Стратегия, Разведка, Производство, Продажи).
3. **Сущности:** Конкретные агенты с узкими ролями внутри блоков.
4. **Связи (Connections):** Потоки данных и результаты между сущностями.
   
   ## 2. Целевая область
   
   Продвижение и продажа **цифровых продуктов** (SaaS, мобильные приложения, подписки) в нескольких нишах (например, астрономия, финансы). Основной упор на скорость генерации трафика, A/B-тестирование креативов и мгновенную обработку возражений.

---

## 3. Архитектура Агентов (Сущности)

### Блок А: Стратегия и Управление (Мозг)

* **AI-Директор (Оркестратор):** Хранитель контекста продуктов. Формирует Markdown-инструкции, ставит цели и делает финальный апрув всех материалов перед публикацией.
* **Стратегический Аналитик:** Собирает данные о конверсиях и стоимости привлечения (CAC). Корректирует курс Директора на основе цифр.
  
  ### Блок Б: Разведка (Данные)
* **Агент-Искатель (Scraper):** Парсинг сторов, профильных форумов, Reddit и конкурентов.
* **Агент-Фильтр (Смысловик):** Выделяет «боли» и инсайты из сырых данных, упаковывая их в тезисы для рекламы.
  
  ### Блок В: Фабрика (Производство)
* **Агент-Промптер:** Переводит стратегию в жесткие ТЗ для генераторов контента.
* **Агент-Копирайтер:** Создание мультиязычных текстов и ASO-описаний.
* **Агент-Дизайнер:** Генерация визуальных ассетов (баннеры, скриншоты интерфейсов).
  
  ### Блок Г: Продажи (Поле)
* **Агент-Паблишер:** Автоматическая публикация контента в рекламные кабинеты и соцсети.
* **Агент Пресейла (SDR):** Мгновенная обработка входящих запросов на лендингах, снятие барьеров перед покупкой/подпиской.

---

## 4. Роадмап реализации (План)

### Этап 1: Подготовка среды (Infrastructure)

* Развертывание инстанса **n8n** через Docker на сервере.
* Настройка реверс-прокси и SSL для работы входящих вебхуков.
* Проверка ресурсов сервера (настройка Swap-файла для стабильности соседствующих проектов).
  
  ### Этап 2: Программирование логики (Core Logic)
* Создание Мастер-воркфлоу (Оркестратора) в n8n.
* Реализация Sub-workflows для каждого блока (Разведка, Фабрика, Сейлз) для модульности и легкой отладки.
* Интеграция Markdown-инструкций в системные промпты агентов.
  
  ### Этап 3: Пилотный запуск (Pilot Campaign)
* Выбор одного цифрового продукта (например, AzerScope или Phi).
* Запуск полного цикла: от поиска «болей» конкурентов до генерации креативов и их публикации.
* Тестирование Агента Пресейла на реальных вопросах пользователей.
  
  ### Этап 4: Масштабирование (Scaling)
* Подключение 2-3 дополнительных продуктов.
* Настройка системы переключения контекста между нишами.
* Автоматизация петли обратной связи (Аналитик -> Директор).
